Postmortem: INC-2008 - file-storage-service

Summary

Severity: SEV1. Duration: 12 minutes. Service affected: file-storage-service.

Root Cause

The incident was caused when a downstream third-party API exceeded its documented rate limit.

Impact

Approximately 1% of requests to file-storage-service returned errors or elevated latency during the incident window.

Resolution

The on-call engineer mitigated by rolling back the most recent deploy and restarting affected pods. Full resolution confirmed via dashboard recovery.

Follow-up Actions

  • Add a regression test covering this failure mode
  • Add an alert threshold to catch this earlier next time
