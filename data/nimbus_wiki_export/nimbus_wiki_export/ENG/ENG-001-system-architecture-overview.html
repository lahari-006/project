System Architecture Overview

Nimbus Cloud Systems runs a microservices architecture on AWS, with services communicating primarily over gRPC internally and REST/GraphQL externally.

High-Level Layout

  • Gateway API - public-facing entry point, handles auth and rate limiting
  • Core services - billing, search, notifications, user-profile (see ENG-011 for the full directory)
  • Data layer - PostgreSQL (transactional), Redis (cache/queues), S3 (blob storage)
  • Event bus - Kafka, used for async workflows (billing events, notification triggers)
