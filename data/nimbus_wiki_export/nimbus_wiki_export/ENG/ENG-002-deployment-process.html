Deployment Process

Current process, effective since the CI/CD migration in Q1 2026. This supersedes the old Jenkins-based process.

Pipeline

All services deploy via GitHub Actions. Merging to main automatically deploys to staging. Production deploys require a manual approval gate from a service owner.


# Trigger a production deploy manually
gh workflow run deploy-prod.yml -f service=billing-service -f version=v2.4.1

Rollback

Rollbacks are one command and take effect within ~90 seconds:


gh workflow run rollback-prod.yml -f service=billing-service -f target_version=v2.4.0

Required Checks Before Merge

  • Unit + integration tests pass
  • Security scan (Snyk) passes
  • At least one approving code review
