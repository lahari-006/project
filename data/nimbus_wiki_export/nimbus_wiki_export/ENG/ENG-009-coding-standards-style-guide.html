Coding Standards & Style Guide

Nimbus engineering standardizes on the following per-language conventions:

  • Python: Black formatting, type hints required on public functions
  • TypeScript: ESLint + Prettier, strict mode enabled
  • Go: gofmt, errors must be wrapped with context

All services require 80%+ test coverage on new code before merge.
